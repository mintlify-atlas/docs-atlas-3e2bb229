## Prerequisites

- **CMake 3.19+** — used for every platform.
- A C++17-capable compiler: MSVC 2019+ on Windows, Xcode 12+ on macOS, or GCC 9+/Clang 10+ on Linux.
- **Git** with submodule support.

## Build the SDK

<Steps>
  <Step title="Clone the full VST 3 SDK">
    `vst3_public_sdk` is a submodule. Always clone the top-level `vst3sdk` repository so that
    CMake can resolve cross-submodule dependencies.

    ```bash
    git clone --recursive https://github.com/steinbergmedia/vst3sdk.git
    cd vst3sdk
    ```

    If you already have a non-recursive clone, fetch the submodules afterwards:

    ```bash
    git submodule update --init --recursive
    ```
  </Step>

  <Step title="Configure with CMake">
    Create a build directory and run CMake. The commands differ slightly per platform.

    <Tabs>
      <Tab title="macOS (Xcode)">
        ```bash
        cmake -GXcode \
              -DCMAKE_BUILD_TYPE=Release \
              -B build .
        ```

        Or generate a Ninja build instead if you prefer the command line:

        ```bash
        cmake -GNinja \
              -DCMAKE_BUILD_TYPE=Release \
              -DCMAKE_OSX_ARCHITECTURES="arm64;x86_64" \
              -B build .
        ```
      </Tab>
      <Tab title="Windows (MSVC)">
        Open a **Developer Command Prompt** and run:

        ```bash
        cmake -G"Visual Studio 17 2022" -A x64 \
              -DCMAKE_BUILD_TYPE=Release \
              -B build .
        ```

        You can also open the generated `build\vst3sdk.sln` in Visual Studio.
      </Tab>
      <Tab title="Linux (GCC / Clang)">
        Install the required packages first:

        ```bash
        sudo apt-get install cmake gcc g++ libx11-dev libxrandr-dev \
                             libxinerama-dev libxcursor-dev libxi-dev
        ```

        Then configure:

        ```bash
        cmake -DCMAKE_BUILD_TYPE=Release \
              -B build .
        ```
      </Tab>
    </Tabs>
  </Step>

  <Step title="Build">
    <Tabs>
      <Tab title="macOS / Linux">
        ```bash
        cmake --build build --config Release -- -j$(nproc 2>/dev/null || sysctl -n hw.logicalcpu)
        ```
      </Tab>
      <Tab title="Windows">
        ```bash
        cmake --build build --config Release
        ```
      </Tab>
    </Tabs>

    Built plug-ins are placed in `build/VST3/Release/` (macOS/Windows) or
    `build/VST3/` (Linux) as `.vst3` bundles.
  </Step>

  <Step title="Understand the AGain sample">
    AGain is the canonical "Hello, World" plug-in. It applies a variable gain to a stereo signal
    and demonstrates every core concept: bus setup, parameter changes, preset state, and
    sample-size negotiation.

    The processor class derives directly from `AudioEffect`:

    ```cpp
    // samples/vst/again/source/again.h
    #pragma once
    #include "public.sdk/source/vst/vstaudioeffect.h"

    namespace Steinberg::Vst {

    class AGain : public AudioEffect
    {
    public:
        AGain ();
        ~AGain () override;

        static FUnknown* createInstance (void* /*context*/)
        {
            return (IAudioProcessor*) new AGain;
        }

        tresult PLUGIN_API initialize (FUnknown* context) SMTG_OVERRIDE;
        tresult PLUGIN_API terminate  ()                  SMTG_OVERRIDE;
        tresult PLUGIN_API setActive  (TBool state)       SMTG_OVERRIDE;
        tresult PLUGIN_API process    (ProcessData& data) SMTG_OVERRIDE;
        tresult PLUGIN_API setState   (IBStream* state)   SMTG_OVERRIDE;
        tresult PLUGIN_API getState   (IBStream* state)   SMTG_OVERRIDE;
        tresult PLUGIN_API setupProcessing (ProcessSetup& newSetup) SMTG_OVERRIDE;
        tresult PLUGIN_API setBusArrangements (
            SpeakerArrangement* inputs,  int32 numIns,
            SpeakerArrangement* outputs, int32 numOuts) SMTG_OVERRIDE;
        tresult PLUGIN_API canProcessSampleSize (int32 symbolicSampleSize) SMTG_OVERRIDE;

    protected:
        float fGain            {1.f};
        float fGainReduction   {0.f};
        float fVuPPMOld        {0.f};
        int32 currentProcessMode {-1};
        bool  bHalfGain        {false};
        bool  bBypass          {false};
    };

    } // namespace Steinberg::Vst
    ```

    `initialize` registers the stereo and event buses:

    ```cpp
    // samples/vst/again/source/again.cpp  (abridged)
    tresult PLUGIN_API AGain::initialize (FUnknown* context)
    {
        tresult result = AudioEffect::initialize (context);
        if (result != kResultOk)
            return result;

        addAudioInput  (STR16 ("Stereo In"),  SpeakerArr::kStereo);
        addAudioOutput (STR16 ("Stereo Out"), SpeakerArr::kStereo);
        addEventInput  (STR16 ("Event In"), 1);

        return kResultOk;
    }
    ```

    `process` reads parameter changes from the host and applies gain:

    ```cpp
    tresult PLUGIN_API AGain::process (ProcessData& data)
    {
        // 1. Read parameter changes (gain, bypass)
        if (IParameterChanges* paramChanges = data.inputParameterChanges)
        {
            int32 numParamsChanged = paramChanges->getParameterCount ();
            for (int32 i = 0; i < numParamsChanged; i++)
            {
                if (IParamValueQueue* paramQueue = paramChanges->getParameterData (i))
                {
                    ParamValue value;
                    int32 sampleOffset;
                    int32 numPoints = paramQueue->getPointCount ();
                    switch (paramQueue->getParameterId ())
                    {
                        case kGainId:
                            if (paramQueue->getPoint (numPoints - 1, sampleOffset, value)
                                    == kResultTrue)
                                fGain = (float)value;
                            break;
                        case kBypassId:
                            if (paramQueue->getPoint (numPoints - 1, sampleOffset, value)
                                    == kResultTrue)
                                bBypass = (value > 0.5f);
                            break;
                    }
                }
            }
        }

        // 2. Guard: nothing to do without I/O
        if (data.numInputs == 0 || data.numOutputs == 0)
            return kResultOk;

        // 3. Apply gain (silence detection and 32/64-bit template helpers omitted)
        float gain = fGain - fGainReduction;
        if (bHalfGain)
            gain *= 0.5f;

        // … buffer copy/processing …

        return kResultOk;
    }
    ```
  </Step>

  <Step title="Register your plug-in with the factory">
    Every VST 3 binary exports a single `GetPluginFactory` function. The macros in
    `source/main/pluginfactory.h` generate this boilerplate for you.

    AGain registers its processor, controller, and a sidechain variant in one entry file:

    ```cpp
    // samples/vst/again/source/againentry.cpp
    #include "again.h"
    #include "againsidechain.h"
    #include "againcontroller.h"
    #include "againcids.h"
    #include "version.h"
    #include "public.sdk/source/main/pluginfactory.h"

    #define stringPluginName         "AGain VST3"
    #define stringPluginSideChainName "AGain SideChain VST3"

    using namespace Steinberg::Vst;

    BEGIN_FACTORY_DEF (stringCompanyName, stringCompanyWeb, stringCompanyEmail)

        // Processor component
        DEF_CLASS2 (INLINE_UID_FROM_FUID (AGainProcessorUID),
                    PClassInfo::kManyInstances,
                    kVstAudioEffectClass,
                    stringPluginName,
                    Vst::kDistributable,
                    AGainVST3Category,
                    FULL_VERSION_STR,
                    kVstVersionString,
                    Steinberg::Vst::AGain::createInstance)

        // Controller component
        DEF_CLASS2 (INLINE_UID_FROM_FUID (AGainControllerUID),
                    PClassInfo::kManyInstances,
                    kVstComponentControllerClass,
                    stringPluginName "Controller",
                    0, "",
                    FULL_VERSION_STR,
                    kVstVersionString,
                    Steinberg::Vst::AGainController::createInstance)

        // Second plug-in: AGain with sidechain
        DEF_CLASS2 (INLINE_UID_FROM_FUID (AGainWithSideChainProcessorUID),
                    PClassInfo::kManyInstances,
                    kVstAudioEffectClass,
                    stringPluginSideChainName,
                    Vst::kDistributable,
                    AGainVST3Category,
                    FULL_VERSION_STR,
                    kVstVersionString,
                    Steinberg::Vst::AGainWithSideChain::createInstance)

    END_FACTORY
    ```

    <Note>
      On Windows you also need a `.def` file that exports `GetPluginFactory`. The SDK's
      CMake scripts add this automatically when you use `smtg_add_vst3plugin`.
    </Note>
  </Step>
</Steps>

## Platform notes

| Platform | Compiler | Output location |
|---|---|---|
| Windows | MSVC 2019+ | `build\VST3\Release\MyPlugin.vst3\` |
| macOS | Xcode / Clang | `build/VST3/Release/MyPlugin.vst3/` |
| Linux | GCC 9+ / Clang 10+ | `build/VST3/MyPlugin.vst3/` |

On macOS the `.vst3` bundle must be signed and notarized before distributing. On Windows, a `.def`
file or a matching module-definition export is required so that `GetPluginFactory` is visible to
the host.
