The VST 3 Public SDK provides everything you need to build professional-grade VST 3 audio plug-ins and hosts. It includes ready-to-use C++ base classes that implement the VST 3 interfaces, so you can focus on your audio processing logic rather than boilerplate.

## Get started

<Steps>
  <Step title="Clone the full VST 3 SDK">
    The public SDK is a submodule of the main SDK repository. Clone it with all dependencies.
    ```bash
    git clone --recursive https://github.com/steinbergmedia/vst3sdk.git
    ```
  </Step>
  <Step title="Configure with CMake">
    Generate your build system using CMake from the SDK root.
    ```bash
    mkdir build && cd build
    cmake ../vst3sdk -DCMAKE_BUILD_TYPE=Release
    ```
  </Step>
  <Step title="Subclass AudioEffect">
    Create your processor by subclassing `Steinberg::Vst::AudioEffect` and override `process()`.
    ```cpp
    #include "public.sdk/source/vst/vstaudioeffect.h"

    class MyGain : public Steinberg::Vst::AudioEffect {
    public:
        tresult PLUGIN_API process(Steinberg::Vst::ProcessData& data) override;
    };
    ```
  </Step>
  <Step title="Register with the factory">
    Export your plug-in using the `BEGIN_FACTORY` / `DEF_CLASS2` / `END_FACTORY` macros.
    ```cpp
    BEGIN_FACTORY ("My Company", "https://example.com", "mailto:info@example.com",
                   PFactoryInfo::kNoFlags)
      DEF_CLASS2 (INLINE_UID_FROM_FUID(MyGain::cid),
                  PClassInfo::kManyInstances, kVstAudioEffectClass,
                  "MyGain", 0, "Fx", "1.0.0", kVstVersionString,
                  MyGain::createInstance)
    END_FACTORY
    ```
  </Step>
</Steps>

<Note>
The VST 3 Public SDK is a submodule of the [VST 3 SDK](https://github.com/steinbergmedia/vst3sdk). Always build from the parent SDK to ensure all dependencies are available. See the [Quick Start](/quickstart) guide for full setup instructions.
</Note>
